# 📩 智联招聘 IM 消息监控与回复 — 使用说明

## 概述

实现智联招聘候选人消息与飞书群的互通，无需手动登录智联网站查看和回复消息。

## 完整工作流程

```
你（飞书群）                     系统自动                   候选人（智联招聘）
    │                              │                            │
    │ 打招呼（手动执行）            │                            │
    │ ────────────────────────────→ 自动搜索+发送招呼语 ────────→ 收到招呼
    │                              │                            │
    │                              │                            │ 回复消息
    │                              │  ←── 每5分钟轮询 ────────── 有新回复
    │                              │                            │
    │ ←── 飞书卡片通知（新消息） ── 检测到增量                   │
    │                              │                            │
    │ 回复：夏旭昊 方便电话沟通吗？ │                            │
    │ ────────────────────────────→ 自动调用API发送 ───────────→ 收到回复
    │                              │                            │
    │ ←── 飞书卡片确认 ✅ ──────── 发送成功                     │
```

---

## 一、在飞书群中操作（日常使用）

### 1. 接收候选人消息通知

系统每 **5分钟** 自动轮询一次（工作日 9:00-18:00），检测到候选人新回复时，飞书招聘组群会收到卡片通知：

```
📩 智联招聘消息提醒
──────────────────────
夏旭昊 | 连接器开发工程师（招应届毕业生）
💬 您好，我对贵司的职位很感兴趣...
🕐 07-01 14:05 | 🔴 1 条未读
──────────────────────
💡 如需回复，请发送：回复 [姓名] [内容]
```

### 2. 回复候选人

在飞书群直接发送：

```
回复 夏旭昊 您好，方便电话沟通一下吗？
```

系统会自动处理并回复确认卡片：

```
✅ 已回复 夏旭昊
📨 内容：您好，方便电话沟通一下吗？
🎯 职位：连接器开发工程师（招应届毕业生）
```

### 3. 查看所有会话

在飞书群发送：

```
查看会话列表
```

系统会列出所有活跃会话及最新消息。

## 二、命令行操作（高级）

### 列出所有会话

```bash
cd /root/.openclaw/workspace-HR-Agent/skills/zhaopin-skill
python3 scripts/im_monitor.py --action list
```

输出示例：
```
📋 会话列表（共 20 个）
  1. 夏旭昊 📅可约面
     职位: 连接器开发工程师（招应届毕业生）
     最后消息: [STAFF] 您好，方便电话沟通吗？
     时间: 07-01 13:48

  2. 王俊权 ⏳待处理 🔴1条未读
     职位: 连接器开发工程师（招应届毕业生）
     最后消息: [USER] 您好，我对贵司...
```

### 查看详细会话信息

```bash
python3 scripts/im_monitor.py --action list --show-all
```

### 手动发送消息

```bash
# 按姓名发送
python3 scripts/im_monitor.py --action send --name "夏旭昊" --content "您好，方便电话沟通吗？"

# 按会话ID发送
python3 scripts/im_monitor.py --action send --session-id "18deeeef..." --content "您好"
```

### 手动轮询检测新消息

```bash
python3 scripts/im_monitor.py --action poll
```

### 查看监控状态

```bash
python3 scripts/im_monitor.py --action status
```

输出示例：
```
📊 IM 监控状态
  已知会话: 20
  上次轮询: 13:50:09
  已处理消息: 0
```

### 快捷回复（推荐）

```bash
python3 scripts/im_reply.py "回复 夏旭昊 您好，方便电话沟通吗？"
```

## 三、打招呼后自动联动

当你使用 `greet_candidate.py` 打招呼后，系统会自动记录到状态文件，后续该候选人的回复会自动被监控到。

```bash
# 打招呼（会自动记录到 IM 状态）
python3 scripts/greet_candidate.py --name "夏旭昊" --keyword "连接器开发工程师"
```

## 四、系统架构

```
┌─────────────────────────────────────────────────────┐
│                  定时任务（每5分钟）                   │
│  cron: */5 9-18 * * 1-5                            │
│  ┌─────────────────────────────────────┐            │
│  │ im_cron.py                          │            │
│  │   ↓                                 │            │
│  │ im_monitor.poll_messages()          │            │
│  │   ↓                                 │            │
│  │ API: POST /api/im/session/list      │            │
│  │ API: POST /api/im/getUnread         │            │
│  │   ↓                                 │            │
│  │ 检测到新消息 → 飞书群卡片通知        │            │
│  └─────────────────────────────────────┘            │
│                                                     │
│  飞书群收到"回复 XXX 内容"                           │
│  ┌─────────────────────────────────────┐            │
│  │ im_reply.py                         │            │
│  │   ↓                                 │            │
│  │ 解析指令 → 查找会话 → 发送消息       │            │
│  │ API: POST /api/im/sendText          │            │
│  │   ↓                                 │            │
│  │ 飞书群卡片确认 ✅                    │            │
│  └─────────────────────────────────────┘            │
└─────────────────────────────────────────────────────┘
```

### 关键文件

| 文件 | 用途 |
|------|------|
| `scripts/im_monitor.py` | 核心监控脚本（轮询/发送/列表/状态） |
| `scripts/im_cron.py` | 定时任务包装脚本 |
| `scripts/im_reply.py` | 回复指令解析与执行 |
| `config/im_state.json` | 消息状态跟踪（自动创建） |

## 五、注意事项

### ⚠️ Cookie 失效
- 如果 Cookie 被踢下线，发送消息会失败
- 需要重新登录智联招聘并更新 Cookie
- 更新后无需重启任何服务，脚本会自动使用新 Cookie

### ⚠️ 消息去重
- 系统通过 `im_state.json` 记录已处理的消息
- 首次部署时，历史未读消息不会重复推送
- 只有**新增**的候选人回复才会触发通知

### ⚠️ 系统消息过滤
- "您的同事XXX正在与候选人沟通"等系统消息会被自动过滤
- 只推送候选人真实回复的消息

### ⚠️ 工作时间
- 轮询仅在 **工作日 9:00-18:00** 进行
- 非工作时间的新消息会在次日 9:00 首次轮询时检测到

## 六、故障排查

### 消息发送失败
```bash
# 检查 Cookie 是否有效
python3 scripts/im_monitor.py --action list

# 如果返回空或报错，说明 Cookie 失效
```

### 未收到飞书通知
```bash
# 手动轮询测试
python3 scripts/im_monitor.py --action poll

# 查看是否有新消息
python3 scripts/im_monitor.py --action status
```

### 找不到候选人会话
```bash
# 列出所有会话确认姓名
python3 scripts/im_monitor.py --action list
# 注意：姓名要完全匹配（含先生/女士）
```
