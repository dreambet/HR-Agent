### 3. 问题修复

#### 问题一：返回候选人数量不足
**现象**：使用 COMPLEX 排序时，API 返回 `foundTotal: 2125` 但 `total: 2`，只抓取到2份而非请求的5份

**根本原因**：
- COMPLEX 排序算法采用多维度加权评分，只返回得分最高的 Top N 结果
- `pageSize` 设置过小

**解决方案**：
```python
# 修复前
payload = {
    "sort": {"type": "COMPLEX", "version": 17},
    "pageSize": count,  # 5
}

# 修复后
payload = {
    "sort": {"type": "TIME", "version": 0},  # 改用最新优先排序
    "pageSize": max(count * 2, 10),  # 确保足够数量
}
```

#### 问题二：educations 参数导致 API 返回异常
**现象**：当 `educations=["1"]` 时，API 返回 0 结果

**根本原因**：单独使用 `["1"]` 时 API 解析异常

**解决方案**：
```python
# 修复前
edu_levels = ["1"]

# 修复后
edu_levels = ["4", "3", "10", "1"]  # 大专、本科、硕士、不限
```

#### 问题三：多余 JSON 文件
**用户要求**：只保存初筛报告 .md 文件，不生成 .json

**解决方案**：移除 JSON 文件生成逻辑
