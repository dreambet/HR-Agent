## 一、初筛报告生成规范（search_resumes.py）

### 1.1 表格格式规范

**问题**：表格分隔行短横线数量必须与表头列数精确匹配

**正确格式**：
- 表头有10列：`|序号|姓名|性别|年龄|工作年限|学历|教育经历|期望薪资|匹配度|备注|`
- 分隔行必须恰好10个短横线：`|-|-|-|-|-|-|-|-|-|-|`

**错误示例**：
- 11个短横线：导致表格在飞书无法正确渲染

**验证方法**：
```python
# 检查分隔行单元格数
separator = '|-|-|-|-|-|-|-|-|-|-|'
cells = [p for p in separator.split('|') if p]
# cells 数量必须等于表头列数
```

### 1.2 简历详情模块字体格式（重要）

**问题**：飞书渲染时加粗和标题会导致字体变大

**最终方案**：使用纯文本格式

```markdown
1. 安先生
基本信息：男，25岁，4年经验，在职-看看机会
求职意向：期望北京，Python，期望薪资2.5万-5万
活跃状态：在线，在线
工作经历：3年 10个月 明亚保险经纪 模型开发工程师
技能标签：算法工程师
简历类型：普通简历，手机可见：
```

**❌ 避免使用**：
- `**加粗**` - 飞书渲染时会导致字体变大
- `### 标题` - 飞书渲染时会导致字体变大

**✅ 正确做法**：
- 使用纯文本格式
- 与普通段落字体大小一致

### 1.3 学历字段数据提取

**问题**：API返回的顶级 `educationLevel` 字段可能为空

**解决方案**：从 `educationExperiences[0]` 中提取

```python
edu_exps = item.get('educationExperiences', [])
if edu_exps:
    for exp in edu_exps[:1]:
        exp_edu_level = exp.get('educationLevel', '')
        if not resume['最高学历'] and exp_edu_level:
            resume['最高学历'] = exp_edu_level
```

## 三、智联招聘API数据提取

### 3.1 Cookie状态检测

- Cookie有效期检测：使用 `/api/talent/search/list` 接口
- Cookie失效特征：返回401或"账号已在其他设备登录"
- 更新频率：建议每天检测一次

### 3.2 搜索API vs 详情API

| 操作 | API接口 | 脚本 |
|------|---------|------|
| 初筛报告 | `POST /api/talent/search/list` | `search_resumes.py` |
| 详细报告 | `POST /api/resume/detail` | `get_resume_detail.py` |

### 3.3 详情API数据完整性

**问题**：详情API可能预加载所有候选人数据，点击卡片时不触发新请求

**解决方案**：缓冲+精确名称匹配
1. 缓冲所有详情API响应
2. 搜索API返回后根据候选人姓名精确匹配
3. 从缓冲中找到目标候选人数据

## 六、常见问题

### Q1: 表格在飞书中不渲染
**原因**：分隔行短横线数量与列数不匹配
**解决**：确保分隔行恰好有N-1个短横线（N=列数）

### Q2: 字体在飞书中显得大
**原因**：使用了 `**加粗**` 或 `### 标题`
**解决**：使用纯文本格式

### Q3: 学历列为空
**原因**：API顶级字段为空
**解决**：从 `educationExperiences[0]` 提取

### Q4: 详细报告数据不完整
**原因**：浏览器预加载导致点击卡片时不触发新请求
**解决**：使用缓冲+精确名称匹配机制
