## 1. pageSize 优化

**修改**：

```python

# 修改后

```

## 2. 动态页数计算

**问题**：之前硬编码 `max_pages=10`，导致可能抓取不完整或浪费资源。

**修改**：
- 根据 `total_count` 和 `pageSize` 动态计算最大页数
- 设置安全上限：最多 100 页

```python
# 计算总页数
calculated_max_pages = (total_count + pageSize - 1) // pageSize  # 向上取整
max_pages = min(calculated_max_pages, 100)  # 最多100页
```

## 3. count 参数修复

**问题**：`--count` 参数没有正确限制返回结果数量，脚本返回了所有找到的简历而不是指定数量。

**修改**：
- 在 `parse_search_response_all` 函数末尾添加结果数量限制
- 同时在分页循环中添加提前停止逻辑

```python
# 限制返回数量不超过 count
if count and len(resumes) > count:
    resumes = resumes[:count]
    print(f"   最终返回前 {count} 条简历")
```

```python
# 分页循环中提前停止
if len(all_results) >= count:
    print(f"   已收集 {len(all_results)} 条结果，达到目标数量 {count}，停止抓取")
    break
```

## 4. 截图功能改为按需启用

**问题**：默认自动截图耗费时间，影响执行效率。

**修改**：
- `get_resume_detail.py` 添加 `--screenshot` 参数
- 默认关闭截图功能
- 仅在需要时显式添加 `--screenshot` 参数

```bash
# 默认不截图（快速执行）
python3 get_resume_detail.py --name "韩先生" --keyword "网络运维工程师"

# 需要截图时
python3 get_resume_detail.py --name "韩先生" --keyword "网络运维工程师" --screenshot
```

**涉及位置**：
- `get_resume_detail.py` 第 48 行：函数签名添加 `screenshot=False`
- `get_resume_detail.py` 第 357-358、406-407、525-526 行：截图逻辑改为条件判断
- `get_resume_detail.py` 第 1568 行：添加 `--screenshot` 参数

## 6. 新增打招呼脚本

**脚本名称**：`greet_candidate.py`

**功能**：
1. 搜索候选人
2. 找到目标候选人卡片
3. 点击"打招呼"按钮
4. 发送个性化问候消息

**使用方式**：
```bash
python3 greet_candidate.py \
  --name "韩先生" \
  --keyword "网络运维工程师" \
  --message "您好，我们公司正在招聘网络运维工程师岗位，看了您的简历觉得很合适..."
```

**注意事项**：
- 发送消息需要消耗平台"聊天权益"
- 如果账号没有足够的聊天权益，会弹出道具购买对话框
- 需要先购买"搜索聊加油包"才能使用打招呼功能

## 7. 平台限制说明

智联招聘平台的打招呼功能有以下限制：
1. 需要有足够的"聊天权益"或"搜索聊加油包"
2. 主动沟通1位候选人消耗1次搜索聊权益
3. 道具购买后30个自然日内有效
4. 部分特殊候选人不会收到智能追聊

## 关键参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| pageSize | 每页返回结果数 | 50 |
| max_pages | 最多抓取页数 | 100（动态计算） |
| filteringChatted | 是否过滤已沟通候选人 | False |
| count | 最终返回的简历数量 | 用户指定 |
| screenshot | 是否保存截图 | False |
