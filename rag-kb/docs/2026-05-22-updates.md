## 二、重大技术修复

### 2.1 打招呼脚本精确选择器修复（关键）

**问题**：打招呼发错人（用户要求联系何先生，实际发给了秦先生）

**根因**：`[class*="card"], [class*="item"]` 匹配到外层容器 `search-resume-list`，其 textContent 包含所有候选人名字，导致点击了排在前面的秦先生的按钮

**智联招聘真实DOM结构**：
```
search-resume-list (容器 - 包含所有人)
└── search-resume-item-wrap (每个候选人独立wrapper) ← 精确选择器
    └── search-resume-item.resume-item-exp
        ├── 姓名、经历等
        └── 打招呼按钮
```

**修复方案**：替换为精确选择器 `.search-resume-item-wrap`

**关键代码**：
```python
# ❌ 错误：过于宽泛的选择器
cards = page.locator('[class*="card"], [class*="item"]').all()

# ✅ 正确：精确选择器
cards = page.locator('.search-resume-item-wrap').all()
```

**验证**：何先生、赵先生、黄先生、寇先生打招呼均正确

## 五、技术经验总结

### 5.1 CSS选择器精确度原则
- 在不确定DOM结构时，优先使用更精确的类名选择器
- 避免使用 `[class*="xxx"]` 等模糊匹配，容易匹配到容器元素
- 如果一个选择器在测试时有效，就不要换成另一个等效但不完全相同的

### 5.2 Playwright路由拦截器注意事项
- `route.continue_()` 只添加不覆盖：如果原始请求已有 `pageSize:20`，添加 `pageSize:50` 无效
- 需要使用 `data[key] = value` 覆盖已有值
- 但修改请求参数可能导致服务器响应异常，需谨慎

### 5.3 虚拟滚动加载问题
- 智联招聘搜索结果使用虚拟滚动（`div.double-virtual-item`）
- `window.scrollTo()` 可能无法触发虚拟滚动加载
- 需要模拟真实的用户滚动行为或使用 Playwright 的 `scrollIntoViewIfNeeded()`
